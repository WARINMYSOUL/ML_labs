# ML_labs

Лабораторные по обработке аудио, текста, изображений, сегментации и seq2seq. Датасеты и тяжёлые артефакты хранятся локально (игнорируются в Git); в репозитории остаются ноутбуки, код и настройки.

## Структура
- `Audio_Lab6/` — аудио: данные в `Audio_Lab6/data/` (игнор).
- `BBC_News_Lab4/` — суммаризация новостей: сырые данные в `data_raw/` (игнор).
- `ImageProcessing_Lab1/` — базовые операции с изображениями; пример `Nekit.png` игнорируется.
- `ImageSegmentation_Lab2/` — сегментация футболки; датасет в `dataset_tshirt/` (игнор).
- `PerformanceAnalysis_Lab3/` — классификация текста; артефакты и датасет игнорируются.
- `Seq2Seq_Lab5/` — seq2seq; данные и чекпоинты (`data/`, `best_seq2seq.pt`) игнорируются.
- `.vscode/` — настройки рабочего пространства, остаются под контролем версий.

## Как начать
1) Создать и активировать виртуальное окружение (по желанию).  
2) Установить зависимости из ноутбуков/requirements при необходимости.  
3) Положить датасеты в указанные папки — они не попадут в репозиторий.

## Гигиена Git
- `.gitignore` исключает датасеты, модели (joblib/pkl/pt/h5/onnx), виртуальные окружения и кеши.  
- Коммить только код, ноутбуки и конфиги, чтобы репозиторий оставался лёгким.
